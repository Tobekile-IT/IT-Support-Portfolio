Service Desk Simulation 15 – Suspected Account
Compromise
Ticket Information
Reported by: Patrick O'Brien
Department: Support
Priority: High (Security Incident)
Issue Description
User reported an unfamiliar login alert from an unknown location. The user denied performing the
login and was concerned the account had been compromised.
Initial Actions
User had already changed their password before contacting IT.
Investigation
Confirmed user identity with a verification code, reset the password to a strong temporary
password, reset MFA, reviewed group memberships for unauthorized changes, and performed a
full endpoint security scan.
Escalation
Escalated the incident to the Security Team for further investigation after containment steps were
completed.
Business Impact
Potential exposure of an account with access to sensitive financial information.
Skills Practiced
Identity verification, password management, MFA reset, Active Directory review, endpoint security,
incident response, and security escalation.
Lessons Learned
Containment comes before investigation. Secure the account, verify permissions, scan the
endpoint, and escalate suspected compromises to the security team
